# Security

セキュリティに関する基本ルールです。

## Secrets Management

- **API Keys**: APIキーや秘密情報は **絶対に** コードにハードコードしないでください。
- **.env**: 環境変数 (`.env`) を使用し、`.gitignore` に含まれていることを確認してください。
- **Public vs Private**: Astroの `PUBLIC_` プレフィックスがついた環境変数はブラウザに露出することに注意してください。

## Dependencies

- 定期的に `npm audit` を実行し、既知の脆弱性がないか確認することを推奨します。
- 新しいパッケージを追加する際は、メンテナンス状況や信頼性を確認してください。

## Content Security

- 外部からの入力を表示する場合は、XSS（クロスサイトスクリプティング）に注意してください。
- AstroはデフォルトでHTMLエスケープを行いますが、`set:html` ディレクティブを使用する場合は特に注意が必要です。
