# Development Principles

開発全般における基本原則を定義します。

## Documentation Philosophy

**適切な場所に適切なことを書く：**

| 場所               | 書くべき内容 | 説明               |
| :----------------- | :----------- | :----------------- |
| **コード**         | How          | 実装の詳細         |
| **テストコード**   | What         | 期待される振る舞い |
| **コミットログ**   | Why          | 変更の理由         |
| **コードコメント** | Why Not      | 非自明な判断の説明 |

## Refactoring Hierarchy

リファクタリング時は以下の優先順位で改善を検討してください：

1. **State（状態）**: ミュータブルな状態と副作用を最小化する
2. **Coupling（結合）**: モジュール間の依存を減らす
3. **Complexity（複雑性）**: ロジックと制御フローを簡潔にする
4. **Code（コード量）**: コード行数を減らす
